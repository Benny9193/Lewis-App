# Lewis Family Images Directory

## Structure

### `family-members/`
Contains portrait photos for family members. Images should be named using the format:
`{id}-{name-formatted}.jpg`

Examples:
- `1-benjamin-cheever-lewis.jpg` (for Benjamin Cheever Lewis, ID 1)
- `45-burwell-connor-harrison.jpg` (for Burwell Connor Harrison, ID 45)

**Recommended Image Specifications:**
- **Format**: JPG or PNG
- **Full Profile Photos**: 400x500px minimum, portrait orientation
- **Tree Node Photos**: 200x250px minimum, portrait orientation
- **Quality**: High resolution for best display

### `properties/`
Contains images of family properties and institutions. Images should be named using the format:
`{property-name-formatted}.jpg`

Examples:
- `lewis-state-bank.jpg`
- `lewis-spring-house.jpg`
- `bc-lewis-house.jpg`
- `lemoyne-center-for-visual-arts.jpg`
- `tallahassee-museum.jpg`

**Recommended Image Specifications:**
- **Format**: JPG or PNG
- **Dimensions**: 800x600px minimum, landscape orientation preferred
- **Quality**: High resolution for property showcases

### Placeholder Images
The system includes automatic fallback to placeholder images:
- `placeholder-portrait.jpg` - for missing member photos in modal profiles
- `placeholder-portrait-small.jpg` - for missing member photos in family tree
- `placeholder-property.jpg` - for missing property images

## Adding New Images

1. Save images in the appropriate directory
2. Use the naming convention specified above
3. Ensure images are web-optimized (reasonable file size)
4. Test that images display properly in both modal and tree views
